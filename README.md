# Нейронные сети: Классификация авторства и диагностика заболеваний

Комплексный проект для обучения и экспериментов с рекуррентными нейронными сетями (RNN, LSTM, GRU) и одномерными сверточными сетями (CNN) на задачах классификации текстов.

## 📋 Описание

Этот проект включает несколько практических реализаций нейронных сетей для:

- **Определения авторства текстов** - классификация фрагментов текста по 20 русским писателям (Пушкин, Гоголь, Лермонтов и др.)
- **Диагностики заболеваний по симптомам** - классификация 10 категорий заболеваний по описанию симптомов
- **Экспериментов с гиперпараметрами** - анализ влияния размера словаря, длины окна и других параметров на точность модели

## 📚 Основные ноутбуки

### 1. **tf_ultra_pro.ipynb** (Главный проект)

Определение авторства текстовых фрагментов среди 20 русских писателей.

**Этапы:**
- ✅ Загрузка корпуса 20 русских писателей (из Yandex Cloud)
- ✅ Предобработка текстов (очистка, нормализация)
- ✅ Разбиение на фрагменты (скользящее окно)
- ✅ Токенизация и паддинг последовательностей
- ✅ Обучение BiLSTM модели
- ✅ Оценка на тестовой выборке (confusion matrix)
- ✅ Проверка на собственном тексте

**Архитектура модели:**
```
Embedding (128 dims)
    ↓
SpatialDropout1D (0.2)
    ↓
Bidirectional LSTM (32 units)
    ↓
Dense (64 units, ReLU)
    ↓
Dropout (0.2)
    ↓
Dense (20 classes, Softmax)
```

**Параметры:**
- VOCAB_SIZE: 10000
- MAX_LEN: 200 слов
- FRAG_LEN: 200 слов
- BATCH_SIZE: 32

### 2. **tf_pro.ipynb** (Диагностика заболеваний)

Классификация 10 категорий заболеваний по описанию симптомов.

**Классы заболеваний:**
- Аппендицит
- Гастрит
- Гепатит
- Дуоденит
- Колит
- Панкреатит
- Холецистит
- Эзофагит
- Энтерит
- Язва

**Архитектура модели:**
- Embedding + Bidirectional LSTM (64 units)
- Second Bidirectional LSTM (32 units)
- Dense layers с BatchNormalization
- Dropout для регуляризации

### 3. **pytorch_lite.ipynb** (Эксперименты с гиперпараметрами)

Исследование влияния гиперпараметров на точность классификации авторства.

**Эксперименты:**
1. **Влияние размера словаря (VOCAB_SIZE):**
   - 5000, 10000, 20000, 40000
   - При фиксированных WIN_SIZE=1000, WIN_HOP=100

2. **Влияние размера окна (WIN_SIZE, WIN_HOP):**
   - (500, 50), (1000, 100), (2000, 200)
   - При фиксированном VOCAB_SIZE=20000

**Результаты:** Итоги экспериментов сводятся в таблицу с метриками.

## 📊 Ключевые компоненты

### Предобработка текста
```python
def clean_only(text):
    # Оставляем буквы и знаки препинания
    text = re.sub(r'[^а-яА-ЯёЁ\s\.!\?,:;\-]', ' ', text.lower())
    return ' '.join(text.split())
```

### Разбиение на фрагменты (скользящее окно)
```python
def split_into_fragments(text, length=FRAG_LEN, step=STEP):
    words = text.split()
    frags = []
    for start in range(0, len(words)-length+1, step):
        frags.append(' '.join(words[start:start+length]))
    return frags
```

### Токенизация
- Использование `Tokenizer` из TensorFlow Keras
- Преобразование текстов в последовательности индексов
- Паддинг последовательностей до фиксированной длины

## 📈 Ожидаемые результаты

### Классификация авторства (tf_ultra_pro)
- **Точность на тестовой выборке:** 70-85% (зависит от параметров)
- **Лучший результат достигается с BiLSTM архитектурой**

### Диагностика заболеваний (tf_pro)
- **Целевая точность:** ≥ 60% (правильное распознавание 6+ заболеваний)
- **Примечание:** Датасет небольшой, поэтому результаты скромнее

### Влияние гиперпараметров (pytorch_lite)
- **VOCAB_SIZE:** Оптимум обычно 10000-20000
- **WIN_SIZE:** Больший размер окна → лучше контекст, но медленнее
- **WIN_HOP:** Меньший шаг → больше примеров, но корреляция

## 🔧 Оптимизация производительности

Проект использует несколько техник оптимизации:

1. **Меньший размер батша** - вместо 256 используется 32 (в 8 раз быстрее)
2. **Меньший VOCAB_SIZE** - 10000 вместо 20000
3. **Меньший MAX_LEN** - 200 вместо 500 слов
4. **Early Stopping** - остановка обучения при отсутствии улучшений
5. **Class Weights** - балансировка классов при разном количестве примеров
